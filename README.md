# AI Token ROI Analytics Engine

Production-ready Python CLI module for answering:

> Is the money spent on AI tokens profitable compared to the ROI generated by employees?

The engine reads employee JSON Lines plus one company JSON document, computes deterministic ROI metrics, optionally asks Gemini for executive narrative fields, and prints one dashboard-ready JSON object.

## Files

- `main.py` - CLI entry point and input loading.
- `analytics.py` - deterministic employee/company metrics, categories, rankings, chart data, anomaly detection, savings estimates.
- `llm_engine.py` - optional OpenAI-compatible LLM reasoning for summaries, recommendations, and explanations.
- `requirements.txt` - runtime dependencies.

## Install

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Usage

```bash
python main.py --employees employees.jsonl --company company.json
```

This prints the JSON result and writes it to `output.json` by default.
It also writes frontend-ready JSON files to `generated_report/`.

Pretty output for local inspection:

```bash
python main.py --employees employees.jsonl --company company.json --pretty
```

Choose another output path:

```bash
python main.py --employees employees.jsonl --company company.json --output demo_output.json
```

Choose another report directory:

```bash
python main.py --employees employees.jsonl --company company.json --report-dir demo_report
```

Use the extended comparison inputs:

```bash
python main.py \
  --employees employees.jsonl \
  --company company.json \
  --employee-profiles employee_profiles.jsonl \
  --model-usage model_usage.jsonl \
  --features features.jsonl
```

## Frontend Report Folder

Each successful run writes:

- `generated_report/index.json` - file manifest for the frontend.
- `generated_report/output.json` - full analytics output.
- `generated_report/dashboard_data.json` - cards, charts, rankings, and forecast chart series.
- `generated_report/executive_report.json` - situation summary, diagnosis, risk level, action plan, assumptions.
- `generated_report/forecast_scenarios.json` - 12-month `baseline_no_action` vs `with_action_plan`.
- `generated_report/action_plan.json` - action plan items generated by Gemini or deterministic fallback.
- `generated_report/issue_cards.json` - clickable CFO issue cards with one-line problem, impact level, details, and solution.
- `generated_report/comparative_analysis.json` - model, employee background, and feature efficiency comparisons.

Gemini is used as the business analyst: it produces the status, executive brief, diagnosis, action plan, and forecast assumptions. Python remains responsible for deterministic metric and forecast calculations.

Disable LLM calls and use deterministic fallback text:

```bash
python main.py --employees employees.jsonl --company company.json --no-llm
```

## LLM Configuration

Set `GEMINI_API_KEY` to enable LLM-generated executive content:

```bash
export GEMINI_API_KEY="..."
export GEMINI_MODEL="gemini-2.5-flash"
```

The engine uses Gemini's REST `generateContent` endpoint by default:

```bash
export GEMINI_BASE_URL="https://generativelanguage.googleapis.com/v1beta"
```

OpenAI is also supported. If `OPENAI_API_KEY` is set, it is selected by default:

```bash
export OPENAI_API_KEY="..."
export OPENAI_MODEL="gpt-4.1"
```

You can force a provider:

```bash
export LLM_PROVIDER="openai"
```

If no API key is present, the CLI still returns valid JSON using deterministic fallback content.

## Output

The command prints one JSON object with:

- `executive_summary`
- `employee_metrics`
- `analysis_sections`
- `comparative_analysis`
- `issue_cards`
- `rankings`
- `chart_data`
- `insights`
- `recommendations`

All numeric calculations avoid divide-by-zero and tolerate missing values by defaulting to safe zeros.

## Analysis Sections

The output contains `analysis_sections`, designed for frontend tabs:

- `cost_efficiency` - `tokens_per_ticket`, `tokens_per_story_point`, `business_output_per_1k_tokens`.
- `productivity_score` - based on tickets resolved, story points, bugs closed, merge requests.
- `quality_score` - based on reopen rate, clarification rate, and completion speed.
- `roi_score` - estimates business output per token spend.
- `company_level_analysis` - spend control, performance impact, and strategic ROI.

Extended inputs add:

- `comparative_analysis.model_efficiency` - tokens and cost by provider/model, plus best model by use case.
- `comparative_analysis.employee_background_efficiency` - token spend and ROI by seniority, skill, and skill level.
- `comparative_analysis.feature_efficiency` - token spend, cost, and value per feature.
- `issue_cards` - CFO-ready cards with `headline`, `efficiency_impact`, `financial_impact`, `problem_details`, and `recommended_solution`.

## Categories

Employees are assigned one of:

- `high_roi`
- `efficient_user`
- `overspender`
- `quality_risk`
- `low_adoption`
- `average_user`

## Notes

The deterministic engine owns all numerical scoring. The LLM only generates narrative fields and per-employee recommendation language, then its JSON is validated before use. If validation fails, deterministic fallback content is used.
